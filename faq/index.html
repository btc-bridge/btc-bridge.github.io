Frequently Asked Questions

Find answers to frequently asked questions about bridging native Bitcoin and using wrapped BTC assets in Web3 ecosystems.

What is the difference between native Bitcoin and Wrapped Bitcoin?

Native Bitcoin (BTC) operates on the Bitcoin blockchain and cannot run smart contracts. Wrapped Bitcoin (like WBTC) is an ERC-20 compatible token on Ethereum that represents native Bitcoin 1:1, allowing it to be used in DeFi applications.

How long does it take to bridge native BTC?

Bridging native Bitcoin usually takes about 60 minutes. This is because the bridge must wait for 6 confirmations on the Bitcoin blockchain to ensure the deposit transaction is final and cannot be reversed.

Is bridging BTC safe?

Bridges carry various risks, including smart-contract vulnerabilities, custodian insolvency, and oracle exploits. Using decentralized options like tBTC or auditing proof-of-reserves for centralized options like BitGo help mitigate these risks, but users should never risk more capital than they can afford to lose.

What happens to my native Bitcoin when I burn wrapped BTC?

When you burn wrapped BTC, the smart contract on the host chain permanently destroys those tokens. This triggers the vault (managed by a custodian or sMPC node group) to broadcast a transaction releasing the equivalent amount of native BTC from the vault to your Bitcoin wallet.

What are proof-of-reserves?

Proof-of-reserves is a cryptographic auditing technique that proves the custodian actually holds the underlying assets. For WBTC, you can compare the total token supply on smart-contract networks with the balance of the public Bitcoin vault addresses to confirm 100% backing.
