How BTC Bridging Works

The bridging mechanism utilizes a cryptographic lock-and-mint model to ensure that every wrapped token on the host chain is backed 1:1 by native Bitcoin in the vault.

The Minting Process (Lock-and-Mint)

Minting is the process of converting native Bitcoin into a wrapped ERC-20 token on a destination network using a lock-and-mint architecture:

  1. Deposit Address Generation: The user connects their Web3 wallet and requests a unique Bitcoin deposit address generated by the bridge's vault.
  2. Native Transaction: The user sends native BTC from their Bitcoin wallet (e.g. Ledger, Electrum) to the generated deposit address.
  3. Block Confirmation: The bridge monitor waits for the transaction to receive a safe number of block confirmations on the Bitcoin blockchain (typically 6 confirmations, around 60 minutes) to ensure transaction finality.
  4. Token Minting: Once confirmed, the bridge smart contracts automatically mint a matching quantity of wrapped tokens (e.g. WBTC, tBTC) directly to the user's connected Web3 wallet.

The Releasing Process (Burn-and-Release)

When a user wants to retrieve their native Bitcoin, they initiate the burn-and-release process:

  1. Initiate Burn: The user submits a transaction on the host smart-contract network to burn their wrapped tokens, specifying their native Bitcoin wallet address as the destination.
  2. Smart Contract Destruction: The host network contract destroys the wrapped tokens, removing them permanently from circulation.
  3. Vault Release: The custody vault (or decentralized node group) detects the burn event and signs a native Bitcoin transaction releasing the corresponding BTC from the vault to the user's specified wallet.
